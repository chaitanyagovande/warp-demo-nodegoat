## Reachable Vulnerabilities (Action Required)

### 1. `body-parser` v1.18.3 → Fix: v1.20.3
- **CVE-2024-45590** | Severity: HIGH
- **Description:** Denial of Service when URL encoding is enabled. A specially crafted payload can flood the server.
- **Reachability:** `server.js:72` — `bodyParser.urlencoded({ extended: false })` is active and receives all HTTP POST bodies.
- **Fix:** `npm install body-parser@latest`

---

### 2. `marked` v0.3.5 → Fix: v4.0.10+
- **CVE-2017-16114** | Severity: HIGH — ReDoS (1k chars blocks ~6 seconds)
- **CVE-2022-21680** | Severity: HIGH — `block.def` catastrophic backtracking
- **CVE-2022-21681** | Severity: HIGH — `inline.reflinkSearch` catastrophic backtracking
- **Reachability:** `server.js:13,126-129` — imported and exposed as `app.locals.marked`; `app/views/memos.html:31` renders untrusted user memo content via `{{ marked(doc.memo) }}`.
- **Risk:** User-supplied memo text can trigger ReDoS, causing severe latency spikes.
- **Fix:** `npm install marked@latest`

---

### 3. `mongodb` v2.2.36 → Fix: v3.1.13+
- **GHSA-mh5c-679w-hh4r** | Severity: HIGH
- **Description:** Denial of Service — the driver crashes when a collection name is invalid and the DB does not exist.
- **Reachability:** `server.js:11` — `MongoClient` is directly imported and used for all database connections.
- **Fix:** `npm install mongodb@latest`

---

### 4. `swig` v1.4.2 → Fix: No upstream patch available
- **CVE-2023-25345** | Severity: HIGH
- **Description:** Directory traversal vulnerability via `include` or `extends` template tags, allowing arbitrary local file reads.
- **Reachability:** `server.js:9,116,135` — swig is the active HTML templating engine for the entire application.
- **Risk:** If any user-controlled input reaches an `include`/`extends` tag, arbitrary files can be read from the server.
- **Fix:** Migrate to a maintained templating engine (e.g., `nunjucks`, `ejs`). No patch available for swig.

---

### 5. `underscore` v1.9.1 → Fix: v1.12.1 / v1.13.8+
- **CVE-2021-23358** | Severity: CRITICAL — Arbitrary code execution via `_.template()` with user-controlled variable properties
- **CVE-2026-27601** | Severity: HIGH — DoS via `_.flatten()` / `_.isEqual()` on recursive structures
- **Reachability:** `config/config.js:1` — imported as `_`. The `_.template()` function is not directly called in current code, but the package is loaded in every request lifecycle.
- **Fix:** `npm install underscore@latest`
